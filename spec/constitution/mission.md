# Misión

## Qué construimos

Aplicación móvil de control de inventario de productos para demostrar
buenas prácticas de arquitectura Flutter a clientes potenciales.
Permite gestionar productos, registrar movimientos de stock y visualizar
estadísticas de inventario.

Piezas principales:

1. **Gestión de productos** — CRUD completo con campos descriptivos.
2. **Control de stock** — Registro de entradas/salidas con historial.
3. **Estadísticas** — Dashboard con métricas clave del inventario.

## Para quién

- **Clientes potenciales** — Personas que podrían contratarte para
  desarrollar sus apps móviles. Buscan ver tu nivel técnico y capacidad
  de entrega.

## Principios

- **Arquitectura limpia** — Separación clara por features, data/presentation/providers.
- **Código de producción** — No atajos por ser "solo demo". Calidad real.
- **Experiencia de usuario pulida** — Animaciones, validaciones, feedback claro.
- **Documentación viva** — Cada decisión técnica documentada en specs.
- **Testing riguroso** — Unit + widget tests para garantizar fiabilidad.

## Qué NO es

- No es una app de producción para usuarios finales reales.
- No es un CRUD simple: incluye movimientos con historial y estadísticas.
- No es un proyecto con backend/servidor: todo es local con Hive CE.
- No es una app multiplataforma web: solo iOS y Android.
